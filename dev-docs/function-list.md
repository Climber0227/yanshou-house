| # | 功能 | 说明 | 当前状态 |
|---|------|------|---------|
| H1.1 | 楼栋数据展示 | 首页/档案页可查看楼栋列表（1-11#楼），每栋显示户数 | 代码有，待验证 |
| H1.2 | 单元选择 | 选楼栋后展示单元（西单元/东单元等） | 代码有，待验证 |
| H1.3 | 楼层选择 | 选单元后展示楼层（1-12 层等） | 代码有，待验证 |
| H1.4 | 户选择 | 选楼层后展示该层各户（西户/东户），含问题数 | 代码有，待验证 |
| H1.5 | 二维码生成 | 每户生成唯一二维码，Canvas 绘制 | ✅ Canvas 绘制，编译通过，待开发者工具验证 |
| H1.6 | 二维码文字 | 二维码下方显示楼号-单元号-房号等信息 | ✅ 文字展示 |
| H1.7 | 扫码识别 | 微信扫码 → 自动解析户 ID → 跳转户操作中心 | 流程通，待真机验证 |
| H1.8 | 手动选户 | 无二维码时通过楼栋→单元→楼层→户手动定位 | 流程通 |
| H1.9 | 最近访问 | 扫码页记录最近访问的 10 户 | ✅ emit + storage 逻辑写通，待验证 |
| H1.10 | 批量导入楼栋数据 | 管理后台上传 Excel 导入楼栋/单元/层/户 | 未开始（管理后台） |

## 模块二：问题上报

> 需求原文：多级选择（户内观感/实测/公区），支持预设描述或自定义；强制拍照上传，可选录视频、语音备注；自动记录上报人/验收人；支持同类问题多户复选一键上传；支持勾选"无问题"完成验收。

| # | 功能 | 说明 | 当前状态 |
|---|------|------|---------|
| P1.1 | 验收类型选择 | 观感/实测/公区三个 TAB 切换 | ✅ 已完成 |
| P1.2 | 观感预设列表 | 墙面/地面/顶棚/门窗/防水/给排水/电气/防护等 12 类预设 | ✅ Mock 21 项，待对照需求附件核实 |
| P1.3 | 实测预设列表 | 室内净高/净开间/进深/门窗洞口尺寸等预设 | ✅ Mock 6 项，待对照需求附件核实 |
| P1.4 | 公区预设列表 | 楼梯/电梯/走廊/墙面/消防等 15 类预设 | ✅ Mock 10 项，待对照需求附件核实 |
| P1.5 | 问题分类筛选 | 选分类（墙面/地面等）后筛选预设项 | ✅ 已完成 |
| P1.6 | 自定义问题描述 | 输入框手动输入描述 | ✅ 已完成 |
| P1.7 | 强制拍照 | 至少 1 张，最多 9 张，调用相机或相册 | ✅ 已完成 |
| P1.8 | 视频录制（可选） | 录视频上传 | ✅ 已完成 |
| P1.9 | 语音备注（可选） | 录音上传 | ✅ 已完成 |
| P1.10 | 整改期限设置 | 上报时选整改截止日期 | ✅ 已完成 |
| P1.11 | 整改责任人指派 | 选择/自动匹配整改责任人 | ⏸ 等后端接口 |
| P1.12 | 表单校验 | 必填项为空时 Toast 提示 | ✅ 已完成 |
| P1.13 | 提交流程 | 提交→ loading → 成功提示 → 返回上一页 | ✅ 已完成 |

## 模块四：整改流程

> 需求原文：自动/手动派单至整改责任单位负责人；状态流转：待整改→整改中→待复查→已闭环；整改责任人上传整改照片和措施；监理复查确认。

| # | 功能 | 说明 | 当前状态 |
|---|------|------|---------|
| R1.1 | 任务列表 | 分段控件：待整改/整改中/待复查/已闭环 | ✅ 已完成 |
| R1.2 | 按角色默认筛选 | 整改员默认"待整改"，查验员/监理默认"待复查" | ✅ 已完成 |
| R1.3 | 接收任务 | 点击"开始整改"→ 状态变为"整改中" | ✅ 已完成 |
| R1.4 | 提交整改 | 整改后照片（至少 1 张）+ 整改措施 → 提交 | ✅ 已完成 |
| R1.5 | 状态变更为待复查 | 提交后状态自动变为"待复查" | ✅ 已完成 |
| R1.6 | 复查通过 | 查验员/监理复查 → 合格 → 闭环 → "已闭环" | ✅ 已完成 |
| R1.7 | 复查退回 | 不合格 → 退回整改 → 状态回到"整改中" | ✅ 已完成 |
| R1.8 | 电子确认 | 复查合格后签署电子确认 | ⏸ MVP 跳过 |
| R1.9 | 操作时间线 | 问题详情页显示完整操作记录 | ✅ 已完成 |
| R1.10 | 超期提醒 | 超出整改期限的问题红色标记 | ✅ 已完成（Mock 数据包含 isOverdue 字段） |
| R1.11 | 整改权限 | 谁上报的问题谁有修改权限 | ⏸ 等后端接口 |

## 模块五：一户一档

> 需求原文：进入任意一户二维码可查看该户所有历史问题、整改状态、验收合格与否；按附件格式生成电子档案；自动汇总生成结果汇总表。

| # | 功能 | 说明 | 当前状态 |
|---|------|------|---------|
| D1.1 | 户问题全景 | 该户所有问题列表（类型/描述/状态/照片/时间） | 有 |
| D1.2 | 状态色点区分 | 橙=待整改 蓝=整改中 紫=待复查 绿=已闭环 | 有 |
| D1.3 | 户验收状态 | 该户各项验收是否全部完成 | 有 |
| D1.4 | 全部整改完成审核 | 户内问题全部整改完成 → 上报人审核同意 → 验房结束 → 首页进度同步更新 | 未实现 |
| D1.5 | 观感表导出 | 按附件 1 格式导出 Word/Excel | 未开始（管理后台） |
| D1.6 | 实测表导出 | 按附件 2 格式导出 | 未开始（管理后台） |
| D1.7 | 公区表导出 | 按附件 3 格式导出 | 未开始（管理后台） |
| D1.8 | 结果汇总表导出 | 按附件 4/5 格式自动汇总 | 未开始（管理后台） |
| D1.9 | 问题库自动关联 | 上传的问题自动归集到对应楼号和房号问题库，可直接导出无需人工整理 | 未开始 |
