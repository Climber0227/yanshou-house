## 一、项目概要

住宅工程质量分户验收微信小程序。使用 uni-app (Vue 3) 构建，编译到微信小程序平台。

**角色**：查验员（上报+复查）、整改员（整改）、监理（复查确认）、管理员（后台管理）

**核心闭环**：上报时指定整改人（姓名+手机号）→ 整改人接收 → 提交整改结果 → 监理/查验员复查 → 通过闭环 / 退回整改

## 二、技术栈

| 层 | 选型 | 说明 |

| 框架 | uni-app (Vue 3) + Vite | 编译到微信小程序 |

| 图标 | 本地 PNG 图片 | 放在 `static/icons/`，替代 emoji |

## 四、关键设计决策

| 决策 | 方案 | 原因 |
|------|------|------|
| 整改人指派 | 上报时填写姓名+手机号 | 现场需要精准指派责任单位，不等扫码认领 |
| 图标方案 | 本地 PNG 图片（`static/icons/`） | 微信不支持内联 SVG，emoji 风格不统一 |
| 超期提醒 | 等后端做短信/微信模板消息 | 前端弹窗无法有效触达，Mock 层保留检测逻辑 |
| 录音 | 弹窗确认→手动停止/30s 自动停 | 统一体验，防止忘记停止 |
| 角色门禁 | 前端根据 store.user.role 控制按钮可见性 | 最终权限需后端校验 |
| 业主角色 | MVP 范围外，未实现 | project-brief 明确跳过 |

## 五、页面流转

```
Tab 首页 ──→ 问题库 / 待办跳转
Tab 扫码 ──→ 扫码 → 户操作中心
                ├── 问题上报 → inspect.vue（观感/实测/公区）
                ├── 批量上报 → batch-report.vue
                ├── 二维码 → qrcode.vue
                ├── 无问题标记
                └── 整户审核（仅上报人可确认）
                │
                └── 问题列表 → issue-detail.vue
                      ├── 接收整改（pending → rectifying）
                      ├── 提交整改（rectifying → pending_review）
                      └── 复查确认（pending_review → closed / rectifying）

Tab 任务 ──→ 任务列表（待整改/整改中/待复查/已闭环）
Tab 我的 ──→ 个人中心 + 通知 + 切换角色
```

## 六、状态流转

```
pending（待整改，已指定整改人）
  → rectifying（整改中，整改人点击接收后）
    → pending_review（待复查，整改人提交结果后）
      → closed（已闭环，复查通过）
      → rectifying（退回整改，复查不合格）
```

每次流转记录 `_transitions` 数组，支持多次退回→二次整改→二次提交→二次复查。

## 十、技术债

1. **输入框**：微信小程序原生 `<input>` 不兼容 Web `box-sizing`，全局禁止了垂直 padding，用 `padding-left/right` + `height` 替代。后续加 input 必须遵守此规则
2. **uView Plus**：集成但几乎没用，产生大量 Sass deprecation warning（不影响构建）
3. **颜色硬编码**：部分 scoped 样式中仍有直接颜色值（如 `#E8E9F1`），应逐步迁移到 tokens 变量
4. **Sass `@import` deprecated**：来自 uView Plus 内部，每次构建输出 9+ 条警告
