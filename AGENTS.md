# AGENTS.md — house-acceptance (分户验收小程序)

任何时候使用中文，除非用户明确要求英文输出。

## 适配标识

- `@@PROJECT:house-acceptance@@`：分户验收微信小程序
- `@@TRUTH:dev-docs/README.md@@`：项目内部真源入口
- `@@STACK:Uni-app (Vue 3) + uView Plus 3.x@@`：小程序端技术栈（跨平台框架，一套代码编译到微信小程序/H5/App）
- `@@OWNER:小程序端由用户负责，后端 API 由后端老师独立负责@@`
- `@@BOUNDARY:核心链路（扫码→上报→整改→闭环）+ 管理后台，不做数据看板、不做业主端@@`

## 核心原则

- 架构优先、设计优先、真源优先、严格验收。
- 本项目前后端分离：你只负责小程序端和管理后台前端，后端 API 由另一位老师独立负责。
- 禁止在小程序端做权限判断、业务逻辑或数据校验——这些全部由后端 API 负责。
- 禁止补丁式开发：不得用 `if` 临时分支、硬编码、mock 数据冒充真实功能。
- 用户明确否定的概念必须从代码、文档、计划和命名中删除。
- 最小改动不是半截改动。

## 真源优先级

事实冲突时按以下顺序处理：

1. 当前源码、测试、微信开发者工具预览截图、真机测试证据。
2. 本文件 `AGENTS.md`。
3. `dev-docs/README.md` 指向的内部真源索引。
4. `dev-docs/project-brief.md`、`dev-docs/architecture.md`、`dev-docs/api-contract.md`、`dev-docs/acceptance.md`。
5. 需求文档 `分户验收小程序开发建议.doc`。

## 必需工作流

```text
当前真源审计
  -> 推理闸
  -> API 合同对齐（与后端老师确认接口）
  -> Mock 数据层实现（覆盖所有 API 接口）
  -> 小程序页面/组件实现（Vue SFC）
  -> HBuilderX 运行到微信开发者工具 + 真机测试
  -> 针对性验收
  -> 文档回写
  -> git 边界复核
```

## 开发策略：Mock 先行

你**不负责后端开发**（后端由另一位老师负责），因此：
- 前端开发全程依赖 Mock 数据，在 `mock/` 目录中实现模拟服务
- `api/` 层封装请求方法，通过环境变量切换 mock ↔ 真实后端
- 所有 API 合同接口必须有对应的 Mock 实现
- Mock 仅限开发调试，严禁随上线包发布

## 推理闸

编码前至少回答：

- 这个功能属于哪个页面/组件？
- 需要调用哪些后端 API？API 合同是否已定义？
- 是否有现成组件可以复用？
- 权限和数据校验是否由后端负责？
- 空状态、加载态、错误态、网络异常态是否都考虑了？

## 设计规则

- 共享 UI 组件放入 `components/` 目录（`.vue` 单文件组件），不得在页面中重复实现。
- 全局样式 token 定义在 `uni.scss` / `styles/tokens.scss`，页面和组件不得硬编码颜色、字号、间距。
- API 调用统一通过 `api/` 层封装（基于 `uni.request`），不得在页面中直接发起请求。
- 配置和环境变量统一管理在 `config/` 中。
- 使用 uView Plus 组件库作为基础 UI 组件，减少自定义样式。
- 跨平台兼容：编写代码时注意微信小程序和 H5 的差异，条件编译用 `#ifdef` / `#ifndef`。

## 实现规则

- 从 API 合同和组件设计开始，不是从页面 UI 倒推。
- 每个页面必须处理：正常态、加载态、空状态、错误态、网络异常态。
- 禁止在页面中写业务规则判断（状态流转、权限逻辑等走后端）。
- 拍照上传使用 `wx.chooseImage` + `wx.uploadFile`，需处理取消、失败、超时。
- 用户可见文案使用中文，错误提示需让现场工人能看懂。

## 验收规则

- 声称"完成""修好""通过"前，必须提供：微信开发者工具截图或真机截图。
- 每个子阶段完成后对照 `dev-docs/acceptance.md` 逐项检查。
- 真机测试必须覆盖安卓和 iOS。
- warning、lint、TODO 都按缺陷处理。

## 停止并询问的情况

- 需求、当前代码、API 合同或验收结果互相冲突。
- 需要新增或修改 API 合同（需与后端老师确认）。
- 需要改动核心流程或状态流转逻辑。
- 用户需求会损害已有的架构或安全边界。
- 真机测试失败且无法定位原因。

## 停止条件

- 本阶段完成条件：核心闭环链路可完整走通 + 管理后台可做基础管理。
- 进入下一阶段前需要的证据：真机测试截图、API 合同验证通过、导出功能可用。
- 当前未验证项：后端技术栈、微信小程序认证状态、云存储方案。
